Imagens Dinâmicas

O termo "imagens dinâmicas" no desenvolvimento web se refere principalmente a imagens que se adaptam ao dispositivo e à tela do usuário. O objetivo é servir a imagem do tamanho e formato mais adequado para cada situação, o que torna o site mais rápido e eficiente, especialmente em celulares.

Existem duas formas principais de fazer isso em HTML: o atributo <srcset> e a tag <picture>. Porém vamos aprender apenas o método mais eficaz

  • Usando a Tag <picture> (Método Mais Poderoso)

    • A tag <picture> é um contêiner que oferece um controle ainda maior. Com ela, você pode servir imagens completamente diferentes, não apenas em tamanhos diferentes. Isso é útil para

    • Diferentes proporções de imagem: Mostrar uma imagem cortada de forma diferente em celular e desktop.
    • Diferentes formatos: Oferecer um formato de imagem moderno (como .webp) para navegadores compatíveis e um formato mais antigo (.jpg) como alternativa.
      •

    Exemplo:

                        
                        <picture>
                            <source media="(min-width: 1051px)" srcset="midia-imagem/gato-grande.jpg">
                            <source media="(min-width: 651px)" srcset="midia-imagem/gato-medio.jpg">
                            <img src="midia-imagem/gato-pequeno.jpg" alt="Foto gato pequeno">
                        </picture>

Neste caso, o navegador:

    Procurará a primeira <source> compatível.

  • Se a tela for maior que 651px, ele carrega midia-imagem/gato-grande.jpg.
  • Se a tela for entre 301px e 1051px, ele carrega midia-imagem/gato-medio.jpg.
  • Se a tela for menor que 651px, ele usa a imagem na tag <img> como padrão.

Em resumo, usar <srcet> e <picture> é a forma moderna e correta de criar imagens dinâmicas, garantindo que a sua página seja rápida e com a melhor experiência visual em qualquer dispositivo.

Reproduzindo Audio

Para inserir áudios em uma página HTML de forma moderna e sem a necessidade de plugins (como o Flash), a tag ideal é a <audio>. Ela é um contêiner que permite que você exiba um player de áudio completo diretamente na sua página.

Estrutura Básica e Atributos Essenciais

A forma mais simples de usar a tag <audio> é com o atributo <controls>.

  • src: O caminho para o arquivo de áudio.
  • controls: Este atributo é crucial. Ele adiciona os controles de reprodução padrão do navegador (botões de play/pause, volume e tempo), tornando o áudio visível e interativo para o usuário.

Exemplo Básico:

            
                <audio src="caminho/para/meu-audio.mp3" controls></audio>

Melhores Práticas: Compatibilidade com Navegadores

Assim como as imagens dinâmicas, os navegadores não suportam todos os mesmos formatos de áudio. Para garantir que seu áudio funcione para todos os usuários, a melhor prática é usar a tag <source> dentro da tag <audio>, fornecendo múltiplos formatos do mesmo arquivo.

O navegador irá percorrer a lista e usar o primeiro formato que ele for capaz de reproduzir. Os formatos mais comuns são .mp3, .ogg e .wav.

Exemplo com Múltiplos Formatos:

            
                <audio controls>
                    <source src="midia-audio/good-star.mp3" type="audio/mpeg">
                    <source src="midia-audio/good-star.wav" type="audio/wav">
                    <source src="midia-audio/good-star.ogg" type="audio/ogg">
                    <p>Seu navegador não suporta a tag de áudio.</p>
                </audio>

Neste exemplo, o texto dentro da tag <p> é uma mensagem de fallback, que só será exibida se o navegador não for compatível com a tag <audio>.

Outros Atributos Úteis

A tag <audio> possui outros atributos que ajudam a controlar o comportamento da reprodução:

  • autoplay: Faz o áudio começar a tocar automaticamente assim que a página é carregada.
    • Importante: Muitos navegadores bloqueiam o autoplay por padrão para não atrapalhar a experiência do usuário. Use com cautela.
  • loop: Faz o áudio se repetir em um loop infinito quando ele termina.
  • muted: Inicia o áudio no modo mudo. Isso é frequentemente usado com o autoplay para contornar o bloqueio de navegadores.

Em resumo, a tag <audio> é a forma padrão e semântica de incluir áudios. O uso do atributo <controls> e de múltiplas tags <source> garante uma experiência de uso consistente para todos.

Atributo <preload>

O atributo <preload> é usado nas tags <audio> e <video> para controlar como o navegador deve carregar o arquivo de mídia.

Ele é uma ferramenta de otimização de performance e tem três valores principais:

  • <none>: Não carrega o arquivo até que o usuário clique no play. É o mais recomendado para economizar dados.
  • <metada>: Carrega apenas informações do arquivo (como a duração), mas não o áudio ou vídeo em si.
  • <auto>: (Padrão) O navegador baixa o arquivo por completo, mesmo que o usuário não interaja.

Exemplo:

Preload none (o mais recomendado para a maioria dos casos).

            
                <audio src="midia-audio/good-star.mp3" controls preload="none"></audio>

Neste exemplo, o arquivo midia-audio/good-star.mp3 só será baixado se o usuário clicar no botão de play. Isso é ótimo para evitar que seu site gaste os dados do usuário, especialmente em dispositivos móveis. Em resumo, o atributo <preload> é uma ferramenta para otimização de performance, dando a você controle sobre o comportamento de carregamento dos arquivos de áudio.

Reproduzindo Video

A tag <video> é a forma padrão e semântica de incorporar vídeos em uma página HTML, sem a necessidade de plugins externos. A sua estrutura é muito parecida com a da tag <audio>.

Estrutura Básica e Atributos Essenciais

Assim como a tag de <audio>, a tag de vídeo requer o atributo <src> e, para ser funcional, o atributo <controls>.

  • <src>: O caminho para o arquivo de vídeo.
  • <controls>: Este atributo é fundamental. Ele exibe os controles de reprodução do navegador (play, pause, volume, tela cheia, etc.), tornando o vídeo visível e interativo para o usuário.
  • <width> e <height>: Embora o tamanho do vídeo possa ser controlado com CSS, é uma boa prática usar esses atributos para definir as dimensões do player.

Exemplo Básico:

            
                <video src="midia-video/developing.mp4" width="800" height="450" controls></video>

Melhores Práticas: Otimização e Compatibilidade

Para garantir a melhor experiência para o usuário e compatibilidade com todos os navegadores, use os seguintes atributos e tags:

  • <poster>: É um atributo exclusivo para vídeos. Ele define a imagem de capa que será exibida antes que o vídeo comece a tocar. Isso melhora a experiência, pois o usuário vê uma imagem em vez de uma tela preta.
  • <source>: Forneça múltiplas versões do mesmo vídeo em diferentes formatos (como .mp4, .webm e .ogg). O navegador irá usar o primeiro formato que ele for capaz de reproduzir.

Exemplo com Múltiplos Formatos e Imagem de Capa:

                
                    <video poster="midia-imagem/poster-pequeno.jpg" width="640" height="360" controls>
                        <source src="midia-video/developing.mp4" type="video/mp4">
                        <source src="midia-video/developing.webm" type="video/webm">
                        <p> Seu navegador não suporta a tag de vídeo.</p>

Vídeos de Serviços Externos (YouTube, Vimeo)

Se você for inserir um vídeo de plataformas como YouTube ou Vimeo, a melhor e mais fácil prática é usar o código de incorporação (embed) que eles fornecem. Esse código utiliza a tag <iframe>.

xemplo (YouTube):

Para encontrar o código de incorporação, basta clicar em "Compartilhar" e em seguida "Incorporar"

Em resumo, a tag <video> é a ferramenta ideal para vídeos próprios. Já para vídeos hospedados em serviços de terceiros, a tag <iframe> é a solução mais comum e eficiente.
